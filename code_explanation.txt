----------------------------------------------------------------
  EXCEL ФАЙЛЫН БҮТЭЦ (_load_time_rows)
----------------------------------------------------------------

Багана [0] : Эхлэх цаг       (жишээ: 00:01:23)
Багана [1] : Дуусгавар цаг   (жишээ: 00:01:35)
Багана [2] : Хэн ярьж байна  T = Багш, S = Сурагч, C = Анги
Багана [3] : Харилцан ярианы дугаар  (1, 2, 3 ...)
Багана [4] : Текст            (Q1, Q2 ... Q6, QL, QO, QC кодтой)

Боловсруулалт:
  - Цагийг секунд болгоно
  - Q кодыг regex-ээр текстээс олно: re.findall(r'Q[1-6]|QL|QO|QC', text)
  - has_q = Q код байгаа эсэх (True/False)

----------------------------------------------------------------
  T->S->T ДАРААЛАЛ (_build_triplets)
----------------------------------------------------------------

Энэ бол таб 2, 3, 5-ын суурь функц.

Алгоритм:
  1. T мөр + Q код байгаа тохиолдолд T1 (багшийн асуулт) гэнэ
  2. T1-ийн дараагийн S эсвэл C мөрийг S (сурагчийн хариулт) гэнэ
  3. S-ийн дараагийн T мөрийг T2 (багшийн дараагийн яриа) гэнэ

Нэг triplet:
  T1 (багшийн асуулт)
   └─> S  (сурагчийн хариулт)
         └─> T2 (багшийн дараагийн яриа)

Буцаах утгууд:
  q_num    : Асуултын дугаар
  q_codes  : Q код (Q1, Q2 ...)
  support  : Дэмжсэн / Дэмжээгүй (ML ангилал)
  extended : Үргэлжлүүлсэн эсэх (T2-д Q код эсвэл ? байгаа)
  row      : T1 мөрийн бүх өгөгдөл
  s_row    : S мөрийн бүх өгөгдөл
  t2       : T2 мөрийн бүх өгөгдөл

================================================================
  TAB 1: СЕГМЕНТИЙН ХУВААЛТ
================================================================

API endpoint:
  GET /api/time/segments  --> Бүх мөрийн жагсаалт
  GET /api/time/summary   --> 3 сегментийн дүн

3 сегмент хуваах логик:
  max_conv = Хамгийн том Харилцан ярианы дугаар
  Сег 1 : дугаар 1 .. max_conv/3
  Сег 2 : дугаар max_conv/3 .. max_conv*2/3
  Сег 3 : дугаар max_conv*2/3 .. max_conv

Дэлгэц (renderSegPage):
  - Metrics: Нийт тээлт, T тоо, S тоо, C тоо, Q кодын тоо
  - Grouped bar chart: Сег 1/2/3 дахь T/S/C тоо (Chart.js)
  - Хүснэгт: Сег + Хүн + Цаг + Q код + Текст
  - Шүүлтүүр: Сегментээр + Хүнээр

================================================================
  TAB 2: БАГШ / СУРАГЧИЙН ХУГАЦАА
================================================================

API endpoint:
  GET /api/time/teacher-time  --> T->S->T хугацааны дата

Хугацаа тооцоолол (triplet тус бүрт):
  teacher_ask   = T1 мөрийн duration (секунд)
  student_ans   = S мөрийн duration (секунд)
  teacher_think = T2.start - S.end  (багшийн боловсруулах хугацаа)

Нийт статистик:
  teacher_time = Багшийн бүх мөрийн нийлбэр цаг
  student_time = Сурагч + Ангийн нийлбэр цаг
  Харьцаа     = teacher_time / student_time

Дэлгэц (renderTimePage):
  - Metrics: Багш нийт, Сурагч нийт, Харьцаа, Дундаж хариулт, Дундаж боловсруулалт
  - Pie chart: Багш vs Сурагч/Анги хугацааны харьцаа
  - Grouped bar: Асуулт # тус бүрийн сурагч хариулт / багш боловсруулах цаг
  - Хүснэгт: Дэлгэрэнгүй цагийн жагсаалт

Тэмдэглэл:
  50_analysis файлд timestamp байхгүй тул энэ таб ажиллахгүй.
  Зөвхөн DATA_time файлд ажиллана.

================================================================
  TAB 3: БАГШИЙН ДЭМЖЛЭГ
================================================================

API endpoint:
  GET /api/time/support  --> T->S->T дараалал + ML ангилал

Дэмжлэг ангилал (_classify_support):
  ML байгаа бол: SupportClassifier (TF-IDF + Logistic Regression)
    - SUPPORT_POS жишээгээр сургасан: "Маш сайн", "Зөв байна" гэх мэт
    - SUPPORT_NEG жишээгээр сургасан: "Дахиад бодно уу", "Анхааральтай" гэх мэт
  ML байхгүй бол: keyword шалгана

Дэлгэрэлт (_classify_extend):
  True  бол: T2 текстэд Q код байна эсвэл ? тэмдэг байна
  False бол: Дараагийн асуулт байхгүй

4 ангилал (2x2):
  DD = Дэмжиж дэлгэрүүлсэн   (Дэмжсэн  + Дэлгэрүүлсэн)  -- ногоон
  DC = Дэмжиж дуусгасан       (Дэмжсэн  + Дуусгасан)     -- цайвар ногоон
  ND = Дэмжилгүй дэлгэрүүлсэн (Дэмжээгүй + Дэлгэрүүлсэн) -- шар
  NC = Дэмжилгүй дуусгасан    (Дэмжээгүй + Дуусгасан)    -- улаан

Дэлгэц (renderSupportPage):
  - Loading state  : Spinner
  - Error state    : Улаан мессеж
  - Empty state    : "T->S->T дараалал олдсонгүй" мессеж
  - Metrics        : Нийт хос, Дэмжсэн %, Дэмжээгүй %, Дэлгэрүүлсэн %
  - Pie chart      : DD/DC/ND/NC 4 өнгөөр
  - Stacked bar    : Q код тус бүрийн 4 ангилалын давхарласан тоо
  - Хүснэгт        : T->S->T дараалал + badge + итгэлцэл %

================================================================
  TAB 4: BLOOM-ЫН АНГИЛАЛ (BLOOMBERT)
================================================================

API endpoint:
  GET /api/time/bloom-ml    --> Багшийн асуулт тус бүрийн Bloom ангилал
  GET /api/analysis/bloom-ml

ML Pipeline (ml_classify.py):

  Текст
    |
    +--> BloomBertClassifier (XLM-RoBERTa)    [ҮНДСЭН]
    |      1. AutoTokenizer  : Текстийг token болгоно
    |      2. AutoModel      : XLM-RoBERTa ~280MB загвар
    |      3. Mean pooling   : 768 хэмжээст вектор гаргана
    |      4. LogisticRegression : Q1..Q6 ангилна
    |      5. Keyword boost  : BLOOM_KEYWORDS-тэй давхарлавал +35%
    |
    +--> BloomClassifier (TF-IDF)              [НӨӨЦ/FALLBACK]
           1. BLOOM_KEYWORDS : Монгол үгийн үндэс шалгана
           2. TfidfVectorizer: char n-gram (2-4) вектор
           3. Cosine similarity: хамгийн ойр centroid-ыг олно

Bloom-ын 6 түвшин:
  Q1 = Сэргээн санах (Remember)  -- цэнхэр
  Q2 = Ойлгох (Understand)       -- ногоон
  Q3 = Хэрэглэх (Apply)         -- улбар шар
  Q4 = Задлан шинжлэх (Analyze) -- нил ягаан
  Q5 = Үнэлэх (Evaluate)        -- улаан
  Q6 = Бүтээх (Create)          -- цэнхэвтэр ногоон

Auto-label систем:
  collect_autolabel(text, threshold=0.70)
    -> conf >= 0.70 бол autolabel_cache.json-д хадгална
  retrain_bloom(min_new=5)
    -> 5+ шинэ жишээ цугларсан бол загварыг дахин сурна

API endpoint-ууд:
  GET  /api/ml/autolabel-stats  --> Хэдэн өгөгдөл цугларсан
  POST /api/ml/retrain          --> Загварыг дахин сургах

Дэлгэц (renderBloomPage):
  - Loading state : Spinner
  - Error state   : Улаан мессеж
  - Metrics       : Нийт + Q1..Q6 тоо + хувь %
  - Pie chart     : Q1..Q6 өнгөөр, tooltip-д хувь
  - Bar chart     : Q код тус бүрийн тоо, hover-т монгол нэр
  - Хүснэгт      : Шүүлтүүр (Түвшин + Дэмжлэг) + мөрийн тоо

================================================================
  TAB 5: АСУУЛТЫН СҮЛЖЭЭ (D3 FORCE GRAPH)
================================================================

API endpoint:
  GET /api/time/question-chain
  GET /api/analysis/question-chain

Зангилаа (node):
  Нэг T->S->T triplet = нэг зангилаа
  Өнгө = DD/DC/ND/NC ангиллаар

Холбоо (link):
  _chain_links() функц:
  Хэрэв T2 нь дараагийн triplet-ийн T1 бол холбоно
  Жишээ: Асуулт 1-ийн дараа Асуулт 3 гарч ирвэл сум татна

Дэлгэц (renderChainPage) — D3.js:
  - force simulation  : Зангилааг автоматаар байрлуулна
  - drag              : Зангилааг гараар зөөж болно
  - zoom + pan        : Томруулж жижигрүүлнэ
  - hover tooltip     : Мэдээлэл харуулна
  - arrow marker      : Дараалал заасан сум

Зангилааны өнгө:
  Ногоон       = Дэмжиж дэлгэрүүлсэн (DD)
  Цайвар ногоон = Дэмжиж дуусгасан (DC)
  Шар          = Дэмжилгүй дэлгэрүүлсэн (ND)
  Улаан        = Дэмжилгүй дуусгасан (NC)
